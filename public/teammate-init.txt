PHASE 1 — ONE-TIME SSH SETUP
------------------------------------------------------------

Use TodoWrite to track these:
  [ ] Check / generate SSH key
  [ ] Add SSH config entry for `legalos`
  [ ] Print public key + tell user to send to Morne
  [ ] Wait for user to confirm Morne authorized it
  [ ] Test connection

ONE STEP PER MESSAGE. Confirm each before continuing.

Step 1.1 — Detect OS via Bash
  Run: `node -p "process.platform"` if Node is installed.
  Or check shell type: `echo $SHELL` (Mac/Linux) vs `$PSVersionTable` (Windows).
  Win = 'win32', Mac = 'darwin', Linux = 'linux'.
  Make sure SSH is available: `ssh -V` (should be OpenSSH on all
  three modern OSes). If not, on Windows guide them to enable
  "OpenSSH Client" under Optional Features.

Step 1.2 — Check / generate SSH key
  Check if a key already exists:
    Mac/Linux: `ls -la ~/.ssh/id_ed25519.pub 2>/dev/null || ls -la ~/.ssh/id_rsa.pub 2>/dev/null`
    Windows: `Test-Path $env:USERPROFILE\.ssh\id_ed25519.pub`

  If neither exists, generate (no passphrase — keeps Claude's
  workflow fluid):
    ssh-keygen -t ed25519 -N "" -f ~/.ssh/id_ed25519 -C "<user-name>@legalos"

  Tell the user: "I generated you an ed25519 SSH key at ~/.ssh/id_ed25519."

Step 1.3 — Add SSH config entry
  Append to ~/.ssh/config (create the file + dir if needed; chmod 600):

    Host legalos
        HostName 51.81.202.161
        User root
        IdentityFile ~/.ssh/id_ed25519
        StrictHostKeyChecking accept-new

  Idempotent: check if a `Host legalos` block already exists; if so,
  leave it alone.

Step 1.4 — Print public key + hand off to Morne
  Run: `cat ~/.ssh/id_ed25519.pub` (or .pub for whichever key exists).

  Tell the user verbatim:

    "Copy the entire `ssh-ed25519 ... <name>@legalos` line above and
     send it to Morne via Signal (1Password if you have it). Tell him:
     'Please add this to authorized_keys.' He'll paste it onto the
     server in about 30 seconds. Tell me when he confirms."

  STOP. Wait for the user to confirm. Do NOT proceed.

Step 1.5 — Test the connection
  Once the user confirms Morne authorized them:
    ssh -o BatchMode=yes -o ConnectTimeout=5 legalos true

  Exit code 0 = success. Tell the user: "Connection works — you're
  ready to edit."

  If it fails:
    - The user may have mistyped the key when sending to Morne. Re-print
      with `cat ~/.ssh/id_ed25519.pub` and ask them to re-send.
    - Or Morne hasn't actually authorized yet — ask them to ping again.

End of Phase 1. Mark TodoWrite items complete.

PHASE 2 — EDITING (the rest of every session)
------------------------------------------------------------

The user will ask for changes in plain language. Examples:
  "Change the home page heading to 'X'"
  "Make the primary buttons darker"
  "Add a new section about pricing"

For every change, the rules are:

  ! NEVER USE Read/Edit/Write/Glob/Grep TOOLS — they operate on the
    user's local PC, which has NO project files. They will silently
    do nothing useful.

  ALWAYS use the Bash tool with SSH commands targeting the alias
  `legalos`. Examples:

    # Read a file
    ssh legalos cat /var/www/vhosts/legenex.com/mo.legenex.com/src/path/file.tsx

    # Search the codebase
    ssh legalos grep -rn 'searchterm' /var/www/vhosts/legenex.com/mo.legenex.com/src

    # List a directory
    ssh legalos ls /var/www/vhosts/legenex.com/mo.legenex.com/src/components

    # Edit (small in-place change with sed)
    ssh legalos sed -i 's|<oldtext>|<newtext>|' /var/www/vhosts/legenex.com/mo.legenex.com/<path>

    # Edit (full file rewrite via heredoc — use single-quoted 'PAYLOAD'
    # so local shell doesn't interpolate $vars or backticks)
    ssh legalos "cat > /var/www/vhosts/legenex.com/mo.legenex.com/<path>" <<'PAYLOAD'
    <new file contents go here verbatim>
    PAYLOAD

After each edit, say to the user:

  "Live at https://mo.legenex.com in ~2 seconds. Hard-refresh (Ctrl+Shift+R
   on Windows / Cmd+Shift+R on Mac) to see it."

If the user reports the change didn't show up, check the dev server
log for compile errors:

    ssh legalos 'journalctl -u legalos-dev -n 40 --no-pager'

If the dev server is stuck, restart it (slow path of last resort):

    ssh legalos systemctl restart legalos-dev
